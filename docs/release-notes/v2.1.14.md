## English

### Codex doc management revamped — 4 sub-tabs (Agents / Memories / Skills / MCP)

The Codex page (Sidebar → Codex) is rebuilt. The legacy marker-managed-block flow is dropped in favor of 4 sub-tabs, each editing the corresponding codex resource in raw mode with independent history keyed by SHA-256 path hash (no cross-tab interference).

- Agents tab: raw read/write `AGENTS.md` at any path, with Tauri file-system picker plus paste-absolute-path. `.git/` detection auto-classifies project-root vs subdir, surfaced as green / orange chips; the default entry is `~/.codex/AGENTS.md` (global blue chip). Path-classification design borrowed from `borawong/AiMaMi`.
- Memories tab: fixed two entries — `~/.codex/memories/MEMORY.md` (primary index) and `memory_summary.md` (auto summary) — the only two user-editable indexes codex actually injects into prompt on session startup. Based on a deep dive of codex's `memories/` crate, `raw_memories.md` / `rollout_summaries/` / `phase2_workspace_diff.md` etc. are Phase 1-2 internal artifacts and are intentionally not exposed for editing.
- Skills tab: scans `~/.codex/skills/<name>/SKILL.md`, each skill name shown as a green chip for raw editing. The "Open folder" button shells out to `open` / `xdg-open` / `explorer`, letting users edit non-SKILL.md companion files (scripts / examples / templates) in Finder / Explorer. codex has no static skill-index file (the skill list is built at runtime into the prompt — see codex `memories/read/src/usage.rs` skills context budget), so no synthetic "directory index" entry is introduced.
- MCP tab — structured rewrite: `toml_edit::DocumentMut` round-trips `~/.codex/config.toml`, modifies only the `[mcp_servers.*]` section, fully preserving the user's other sections, comments, and key ordering. Frontend is a left list (with local / remote chips) and a right JSON read-only / textarea toggle, with two bottom buttons (Add / Edit). `upsert_server` does read-modify-set so unmodeled fields (`tools` per-tool approval map / `env_vars` array / future codex schema additions) are preserved rather than silently dropped. The Plugins sub-tab scans `~/.codex/plugins/cache/<market>/<plugin>/<version>/` with enable toggle and uninstall (clears cache dir + removes the toml entry, with double confirmation). Marketplace and Deeplink (`codex-app-transfer://v1/import?...` URL scheme + confirmation modal) are fully wired on the backend, but the frontend entries are hidden behind followup #40 until `Cmochance/codex-app-transfer-registry` is seeded with curated presets.

Devin pre-merge fixes (13 in total): tarball download wrapped in a 60s timeout plus Content-Length pre-check and streaming size cap to defeat OOM attacks; plugin name / marketplace / version each get full-string rejection of `.` / `..` plus char-class validation against `../` traversal; uninstall enforces the same path-safety; restore paths use atomic tmp+rename to avoid partial files on crash; tarball wrapper-flatten avoids the false-positive collision when an inner child shares the wrapper's basename (by moving the wrapper out of staged first); `InstallInput` gains `#[serde(rename_all = "camelCase")]` so plugin install no longer fails on `tarballUrl`; modal placement consistency, etc.

Followup: #40 marketplace UI entry pending registry repo; #41 `config.toml` concurrent RMW with `fs2` lock (theoretical race, no realistic trigger today, kept for future real-world report).
