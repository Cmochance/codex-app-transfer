## English

### `apply_patch` diff UI now works on chat-completions providers (closes #235)

Full fix for the missing diff UI when Codex App calls the `apply_patch` tool on chat-completions providers (DeepSeek / Kimi / Xiaomi MiMo, etc.). After the fix, all six diff UI elements render correctly: line numbers, `+N -M` add/remove counts, file list, color highlighting, tool-call header, and clickable file names.

Effective path: Codex App emits an `apply_patch` call via the Responses API `custom_tool_call` form; the adapter bridges it bidirectionally to the chat-completions `function_call` form; the upstream chat-completions model produces a V4A-format patch text; the adapter rewrites the SSE stream back to the Responses protocol so Codex App can render the diff UI.

Changes:

- Wire-layer `custom_tool_call` SSE bridging + multi-turn `previous_response_id` history replay (PR #236)
- Prompt fixes: V4A `@@` single-sided syntax / removed misleading EMPTY LINE anchor advice / explicit MINIMAL Update form (`-line` / `+line` without `@@`) / Add File requires `+` prefix on every line / no space between prefix and content / `*** Begin Patch` as the literal first line / Update + Move requires ≥1 hunk (use Delete + Add File for pure rename) (PR #236 + PR #240)
- Envelope `output[]` interrupted `apply_patch` status matches the streaming `done` event (prevents partial V4A from being treated as complete; Devin pre-merge review BUG fix)
- Guidance system message injected only on the first turn, preventing accumulation across multi-turn conversations (Devin pre-merge review BUG fix)
- README adds a top-of-page test coverage notice (providers verified end-to-end vs. providers still needing real-world regression + contact for API-key-loaning testers)
- README adds a macOS signing notice (Apple Developer ID + Notarization + Gatekeeper terminology + `right-click → Open` workaround)

Upstream borrowing: [`openai/codex`](https://github.com/openai/codex) `core/src/codex.rs` apply_patch protocol structure + V4A patch format, Apache-2.0. See [ACKNOWLEDGEMENTS.md](../../ACKNOWLEDGEMENTS.md).
