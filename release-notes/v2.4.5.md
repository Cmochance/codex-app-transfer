## English

### Full Grok Build provider integration + Codex 26.707 adaptation

This release completes the xAI Grok Build provider integration — login, conversation, tool calls, quota — and adapts to Codex desktop 26.707's app rename and new Quick Chat.

- Grok Build integration (#590): the xAI grok CLI coding backend `cli-chat-proxy.grok.com/v1`, OpenAI Responses passthrough with client fingerprint headers; no local grok CLI required.
- Login via authorization-code + PKCE (#594): Cloudflare blocked the device flow; with a browser consent page + local loopback callback, any account can log in.
- apply_patch / `tool_search` closure (#592): request-side tools conversion, a response-side SSE shim and input-history rewriting — grok can edit files and discover deferred MCP tools.
- Weekly quota + grok-4.5 (#595): the usage panel shows weekly quota and reset time; after upstream retired grok-build, the default model is grok-4.5 (500k context, three reasoning-effort tiers).
- 26.707 rename adaptation (#596): after Codex.app→ChatGPT.app, process matching verifies ownership by bundle id; restart and login work across old and new installs.
- Quick Chat with custom models (#598): Quick Chat forwards through the proxy to custom providers, multi-turn context included; the relabeled model picker matches actual invocation.

Verification: adapters tests and registry golden fixtures pass, `cargo fmt --check` is clean; real-machine e2e covers grok login / conversation / quota / apply_patch, the 26.707 restart kill and Quick Chat relabel.
